# Founder Workflow Audit — Delivery Template

This template structures the audit output delivered to each customer after intake form submission.

---

## Structure

### 1. Executive Summary (1 page)

**The Recommendation:**
- Primary workflow to automate (name + one-liner)
- Hours/week saved (estimated)
- Implementation difficulty (Easy / Medium / Complex)

**Why this wins:**
- Frequency score (1-10)
- Pain score (1-10)
- Feasibility score (1-10)
- Total score (weighted average)

### 2. Rejection Memo

**Workflows considered but NOT recommended:**
- Brief reason each was cut
- What would need to change to reconsider

### 3. Implementation Blueprint

#### Phase 1: Quick Win (Week 1)
- Specific prompt or automation step
- Tools required
- Expected output quality

#### Phase 2: Refinement (Week 2-3)
- Iteration path
- Escape valves / fallbacks

#### Phase 3: Confidence (Month 2+)
- Metrics to track
- Signals to expand

### 4. Assets Included

- [ ] Primary prompt (Markdown)
- [ ] 2-3 variants with personality notes
- [ ] Automation logic (if applicable)
- [ ] Quality checklist for outputs
- [ ] Hand-off instructions (if VA-assisted)

### 5. Deliverable Format

- Single PDF export OR
- Notion/doc page with share access
- Loom video walkthrough (optional, if complex)

---

## Quality Gates

- [ ] All intake questions answered
- [ ] At least one alternative explicitly rejected with rationale
- [ ] Prompt testable in customer's existing tools
- [ ] Hand-off path clear for non-technical
